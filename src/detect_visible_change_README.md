## 2. 问题概述

### 2.1 问题描述

在对原始RMSteg进行批量测试时，发现部分图片在隐写后出现了肉眼可见的明显变化，违背了隐写的不可感知性要求。

### 2.2 问题图片统计

| 统计项 | 值 |
|--------|-----|
| **测试图片总数** | 51张 |
| **问题图片数** | 14张 |
| **问题占比** | 27.45% |

## 3. 检测方法

### 3.1 检测指标

| 指标 | 全称 | 正常范围 | 说明 |
|------|------|---------|------|
| **SSIM** | Structural Similarity Index | &gt; 0.92 | 结构相似性，越接近1越好 |
| **PSNR** | Peak Signal-to-Noise Ratio | &gt; 30 dB | 峰值信噪比，越高越好 |
| **MaxDiff** | 最大像素差 | &lt; 0.1 | 像素级最大变化量 |
| **Perceptual** | 感知差异 | &lt; 0.1 | 基于直方图的感知差异 |

### 3.2 判定规则

采用评分系统进行判断：

| 规则 | 说明 |
|------|------|
| 每个指标超出范围 | +1分 |
| 总分 ≥ 2分 | **检测到明显变化** ⚠️ |
| 总分 &lt; 2分 | **隐写效果良好** ✅ |

## 5. 原因分析

### 5.1 问题图片的共同特征

通过对问题图片的观察分析，发现以下共同特征：

| 特征 | 说明 |
|------|------|
| **高纹理区域** | 图片本身纹理丰富，隐写修改更容易被察觉 |
| **明亮区域** | 明亮区域像素变化更明显 |
| **特定内容** | 某些特定内容（如人脸、文字）对修改更敏感 |

### 5.2 模型层面分析

从模型角度分析可能的原因：

1. **固定尺寸缩放**：原始模型强制缩放图片到224×224，可能引入额外失真
2. **特征不匹配**：部分图片的特征与训练数据分布差异较大
3. **注意力机制偏差**：AttnFlow在某些区域的注意力分配不合理

## 6. 使用指南

### 6.1 工具使用

#### 6.1.1 检测单张图片

```bash
python detect_visible_change.py --image ./test_img/1.png
```

如果已有隐写后图片：

```bash
python detect_visible_change.py --image ./test_img/1.png --steg ./all_test_result/1_steg.png
```

#### 6.1.2 批量检测

```bash
python detect_visible_change.py --dir ./test_img --output my_results.csv
```

默认检测所有图片：

```bash
python detect_visible_change.py
```

## 7. 建议与展望

### 7.1 改进建议

1. **自适应尺寸**：考虑使用自适应分辨率版本（`auto_224model/`）
2. **特殊样本优化**：针对问题图片设计专门的优化策略
3. **动态阈值**：根据图片内容自适应调整检测阈值

### 7.2 未来方向

1. **鲁棒性研究**：深入研究为什么这些图片更容易产生明显变化
2. **新算法设计**：设计针对高纹理区域的专门隐写算法
3. **数据集构建**：构建更有代表性的数据集
