## 1. 概述

本报告是 RMSteg 自适应分辨率版的完整使用说明。

### 1.1 核心改进

| 方案 | 是否有效 |
|------|---------|
|  简单上采样/下采样 | 无效 |
|  图像金字塔算法（5层） | 有效 |

### 1.2 算法流程

1. **构建金字塔**：将任意分辨率图像分解为5个尺度
2. **多尺度处理**：在每个尺度上应用原始 RMSteg 网络
3. **融合结果**：将多尺度结果融合，恢复原始尺寸
4. **保持基线**：原始 RMSteg 算法不变

### 1.3 训练设置

| 项目 | 原始版本（src） | 自适应版本（auto_224model） |
|------|----------------|------------------------------|
| **训练轮数** | 30 | 30（一致） |
| **金字塔层数** | - | 5 |
| **预训练权重** | rmsteg.pth | 支持加载原始预训练权重 |
| **输入尺寸** | 224×224 | 任意分辨率 |

## 3. 快速开始

### 3.1 单图测试（推荐）

```bash
cd auto_224model
python final_test.py
```

特点：
- 自动在 `./test_img/` 寻找图片，默认使用 `test2.png`
- 保持宿主图片原始尺寸
- QR码自适应匹配
- 会尝试加载原始 RMSteg 预训练权重
- 结果保存到 `./result/`

### 3.2 批量测试

```bash
python all_test.py
```

特点：
- 批量处理 `./test_img/` 下的所有图片
- 结果保存到 `./result/`

### 3.3 评估模型

```bash
python evaluate.py
```

特点：
- 定量评估：SSIM、PSNR、LPIPS
- 结果保存到 `./result/`

### 3.4 训练模型

```bash
python train_single_gpu.py
```

特点：
- 训练轮数：30轮（与原始一致）
- 金字塔层数：5层
- 训练集：`./train_img/`
- 保存最终权重
- TensorBoard 日志
- 无需分布式设置

## 4. 各脚本功能详解

### 4.1 final_test.py

**功能**：单图完整测试（编码→失真→解码）
**特点**：
- 保持宿主图片原始尺寸
- QR码自适应匹配
- 使用5层图像金字塔算法
- 不使用简单上采样/下采样
- 严格对齐原始 RMSteg 的输出格式
- 结果文件名：`test_*.png`

### 4.2 all_test.py

**功能**：批量测试所有图片
**特点**：
- 处理 `./test_img/` 下的所有图片
- 每个图片生成完整结果
- 结果保存到 `./result/`

### 4.3 evaluate.py

**功能**：评估模型性能
**指标**：
- SSIM：结构相似性，越接近 1 越好（&gt; 0.98）
- PSNR：峰值信噪比，越大越好（&gt; 35 dB）
- LPIPS：感知相似性，越小越好（&lt; 0.05）

### 4.4 train_single_gpu.py

**功能**：单卡训练
**特点**：
- 使用5层图像金字塔算法
- 训练轮数：30轮（与原始一致）
- 保存最终权重到 `./checkpoints/`
- TensorBoard 日志到 `./log/`
- 无需分布式设置

## 5. 自适应原理详解

### 5.1 为什么不使用简单上采样/下采样无效

1. 缩放过程会丢失信息
2. 引入额外伪影
3. 没有真正解决固定分辨率限制问题

### 5.2 为什么图像金字塔有效？

优点：
1. **多尺度处理**：在多个尺度上处理，保持细节
2. **完全不变**：原始 RMSteg 网络完全不变
3. **任意分辨率**：理论上支持任意尺寸
4. **算法成熟**：图像金字塔是成熟的多尺度处理方法
5. **保留原始特性**：保留 IQRT、AttnFlow、失真训练等所有原始特性

### 5.3 核心代码实现

**文件位置**：`auto_224model/model/attnflow_adaptive.py`

**关键函数**：
| 功能 | 说明 |
|------|------|
| `build_pyramid` | 构建5层金字塔 |
| `fuse_pyramid` | 融合金字塔，恢复原始尺寸 |
| `encode` | 编码过程（严格对齐原始） |
| `decode` | 解码过程（严格对齐原始） |
| `forward` | 完整前向（编码→失真→解码） |
| `calc_loss` | 损失计算（严格对齐原始） |

## 10. 注意事项

1. **金字塔层数**：当前使用 5 个尺度，可以根据需要调整
2. **大尺寸图片**：大尺寸图片会生成更多尺度，速度稍慢
3. **内存**：处理超大图片时注意内存使用
4. **环境**：需要和原始版本相同的依赖
5. **src文件夹**：完全不修改，保持原始状态
